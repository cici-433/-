# ShortTV Android 项目总结（3 点记忆版）

> 目标：把原来的内容压缩成更适合记忆和面试表达的结构。全篇都按 3 点展开，每段最多 3 个重点。

## 一、先记住这 3 句话

- **项目本质**：这是一个海外短剧 Android 项目，核心不是单纯播放视频，而是内容分发、沉浸式播放和商业化变现的完整产品。
- **技术本质**：项目以 Activity/Fragment + ViewModel + Repo 分层为主，播放器、广告、支付、推送、配置等基础能力做了统一封装。
- **面试本质**：你要讲的重点不是页面有多少，而是播放链路、商业化链路、增长和工程化能力。

## 二、项目定位：只记 3 个关键词

- **内容消费**：首页、搜索、榜单、分类、剧库、继续观看，负责把内容分发给用户。
- **沉浸播放**：Shorts 流和沉浸页负责连续切集、上下滑切换、首帧体验、广告插入和任务联动。
- **商业化增长**：订阅、金币、广告解锁、推送、Deep Link、活动页一起组成变现和拉活闭环。

## 三、业务结构：只分 3 条线

- **内容线**：Discover、Search、榜单、分类、预览播放、继续观看，解决“让用户看到什么”。
- **播放线**：ShortsFragment、ImmersionActivity、下一部推荐、倍速、进度、缓冲、内容评级，解决“让用户如何持续看下去”。
- **经营线**：Rewards、Subscription、金币充值、FCM 推送、活动资源位，解决“如何变现和召回用户”。

## 四、架构重点：只讲 3 层意思

- **页面层**：以 Activity + Fragment + DataBinding/ViewBinding 为主，页面只负责交互和展示。
- **状态与数据层**：大量页面接 ViewModel，数据按 LocalDS + RemoteDS + Repo 组织，便于状态流转和数据复用。
- **基础设施层**：网络、播放器、广告、支付、推送、归因、配置等能力统一收口到 manager/repo/startup 体系。

## 五、项目亮点：只讲 3 个就够了

- **亮点 1：播放器和沉浸式播放做得深**
  - 不只是接播放器 SDK，而是围绕短剧场景做了加密播放、ABR 起播选档、预加载、缓存、首帧优化、CDN/HTTPDNS 等能力。
  - 真正体现的是连续播放体验优化，而不是“视频能播就行”。
- **亮点 2：商业化不是单点，而是三线并行**
  - 广告、Google Play 订阅、金币充值同时存在，还要支持广告解锁、奖励币、补单恢复、订阅奖励等组合场景。
  - 这能体现你理解的是完整商业模型，不只是客户端页面。
- **亮点 3：配置化和增长闭环比较强**
  - 项目通过 CMS、Firebase Remote Config、A/B 实验动态控制逻辑，同时接了 FCM、Deep Link、Campaign、活动落地页。
  - 说明这个项目支持精细化运营、灰度试验和用户召回，不是只能靠发版改逻辑。

## 六、工程化成熟度：只记 3 件事

- **启动治理**：使用 AndroidX Startup 拆分初始化，避免所有逻辑堆到 Application。
- **数据治理**：Repo 按业务域拆分，网络、数据库、日志、CDN 等能力组织比较完整。
- **交付治理**：有 GitLab CI、自动构建上传、通知分发，说明项目不只是开发功能，也有交付闭环。

## 七、上线能力：只记 3 个维度

- **稳定性**：接入 Crashlytics 和 NDK 崩溃统计，前后台切换时会 flush 事件和日志，关键链路有缓存与恢复机制。
- **性能**：播放器场景做了预加载、缓存、ABR、CDN 切换，启动通过 Startup 拆分，构建层做了混淆和 split。
- **安全合规**：用了 Firebase App Check Play Integrity、networkSecurityConfig、加密剧集接口，并适配 Android 13 通知权限等要求。

## 八、项目最难的点：只答 3 个难点

- **难点 1：播放状态复杂**
  - 长时播放、连续切集、广告插入、小窗/PIP、通知恢复等状态会叠加，不是普通详情页。
- **难点 2：商业化逻辑复杂**
  - 广告、金币、订阅三条线同时存在，会影响解锁路径、展示策略和埋点设计。
- **难点 3：配置和增长链路复杂**
  - 很多行为由远程配置和 A/B 实验控制，同时还要承接推送、Deep Link、Campaign、活动页流量。

## 九、你在这个项目里的价值：只答 3 点

- **业务价值**：我参与的不只是页面开发，而是内容播放、商业化和增长承接这些核心链路。
- **技术价值**：我接触的是沉浸页、播放器能力、支付广告推送、配置化控制等复杂场景。
- **工程价值**：我理解项目如何通过 Repo、Manager、Startup、Remote Config、CI 维持长期可维护性。
