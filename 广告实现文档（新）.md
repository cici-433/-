# 广告实现文档（新）

## 1. 广告的场景和用途

本项目将“广告场景”作为产品用途与触发时机的抽象：业务侧选择一个场景（`AdScene`）并传入更细粒度的 `sceneId`（字符串）用于埋点与区分入口；底层再映射到具体广告格式（激励/插屏/开屏/原生/横幅/贴片）以及对应配置（开关、广告单元组、频控）。

### 1.2 场景与 sceneId（字符串）的分工

- `AdScene`：用于选取配置与策略（是否开关、广告单元组、频控/冷却、预加载策略等）。
- `sceneId`：调用 `AdManager.show*` 时由业务传入，用于更细粒度区分入口与埋点（例如“退出沉浸页”“解锁弹窗”“启动页”等）。

## 2. 不看现有实现，如何接入 AdMob 并完成展示

这一节假设你不依赖项目里现成的广告封装/聚合库，也不复用当前的广告实现链路，仅使用 Google Mobile Ads SDK（AdMob 官方 SDK）完成从 0 到 1 的接入与展示。

### 2.1 添加依赖

本项目当前通过封装库间接使用了 `com.google.android.gms.ads.*`（见原生广告布局中的 `NativeAdView/MediaView`），但如果你要“直接接入 AdMob”，需要在 `app/build.gradle` 显式加入 Google Mobile Ads SDK 依赖（以及可选的 UMP 合规依赖）。

### 2.2 配置 AdMob App ID（Manifest + string）

Manifest 里已经预留了 AdMob 的 application id 配置：

- [AndroidManifest.xml:AdMob meta-data](file:///Users/chenxunlin/trace-workspace/shortTV/app/src/main/AndroidManifest.xml#L354-L373)

App ID 的 string 在本项目里放在：

- [third_party.xml:admob_app_id](file:///Users/chenxunlin/trace-workspace/shortTV/app/src/main/res/values/third_party.xml#L3-L5)

发布前将 `admob_app_id` 替换为你自己 AdMob 后台生成的 App ID。

### 2.3 初始化 Mobile Ads（只做一次）

初始化只需要在应用启动后调用一次 `MobileAds.initialize(context)`。在这个工程里比较合适的落点是 AndroidX Startup 初始化链路中（可参考当前的配置初始化入口：[ConfigureInitializer](file:///Users/chenxunlin/trace-workspace/shortTV/app/src/main/java/com/startshorts/androidplayer/startup/ConfigureInitializer.kt#L26-L44)）。

### 2.4 合规（GDPR/CCPA）在“直接接入 AdMob”下怎么做

如果直接使用 AdMob 官方 SDK，建议使用 Google UMP（User Messaging Platform）来处理 GDPR 同意流程：在展示任何广告前，先拉取 consent 信息、必要时展示同意弹窗，再开始加载/展示广告。

本项目现成的“展示前等待合规结果”的思路可以参考：[AdManager.waitGDPRConsent](file:///Users/chenxunlin/trace-workspace/shortTV/app/src/main/java/com/startshorts/androidplayer/manager/ad/AdManager.kt#L1316-L1437)，但对应实现类与调用 API 需要替换为 UMP 的实现。

### 2.5 准备广告位 ID（Ad Unit ID）

从 0 接入时建议先用 AdMob 官方测试 adUnitId 跑通链路，再切换为线上 id。

本项目里已经有一份按场景组织的 AdMob 测试 id（用于开发环境），位置在：

- [assets/ad/adUnitId_android_v3.json](file:///Users/chenxunlin/trace-workspace/shortTV/app/src/main/assets/ad/adUnitId_android_v3.json)

如果你不做配置化，可以先把这些 id 写死成常量；后续再接入 Remote Config 或本地 JSON 都可以。

### 2.6 各广告类型的加载与展示（AdMob 直连）

AdMob 直连时，不同广告类型的调用入口是不同的类：

- 横幅：`AdView`
- 插屏：`InterstitialAd`
- 激励：`RewardedAd`
- 开屏：`AppOpenAd`
- 原生：`AdLoader` + `NativeAd` + `NativeAdView`

其中“原生广告布局”需要 `NativeAdView` 作为根容器，并在代码里把 headline/callToAction/icon/mediaView 等字段绑定到对应 view。项目里已有可复用的 AdMob 原生布局示例：

- [view_native_ad_admob_1.xml](file:///Users/chenxunlin/trace-workspace/shortTV/app/src/main/res/layout/view_native_ad_admob_1.xml)
- [view_native_ad_admob_2.xml](file:///Users/chenxunlin/trace-workspace/shortTV/app/src/main/res/layout/view_native_ad_admob_2.xml)

### 2.7 嵌入式弹框展示（非全屏）

AdMob 官方 SDK 里“激励/插屏/开屏”都是全屏；如果你要“弹框里展示广告内容（非全屏）”，通常做法是：

- 弹框（Dialog/BottomSheet）里放一个容器
- 在容器里展示原生（`NativeAdView`）或横幅（`AdView`）

本项目的原生布局文件已经满足“容器可嵌入”的结构（见上一节的布局引用）。
